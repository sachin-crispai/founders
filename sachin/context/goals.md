# Goals — Sachin

## Current Focus (2026)

- Build CrispAI's core AI infrastructure — the foundational layer before commercial launch
- Identify and validate a niche vertical: Robotics, Construction, or Permit Processing
- Secure 5–10 POC customers to prove product-market fit ahead of acquisition

## Short-Term (next 3–6 months)

- [ ] Finalize core on-prem AI infrastructure stack
- [ ] Sign first 2–3 POC customers in a target vertical (Robotics / Construction / Permits)
- [ ] Define acquisition narrative and positioning for $5M–$10M exit
- [ ] Build automated DAG-based job orchestration with error recovery for CrispAI workflows

## Long-Term Vision (2–3 years)

Position CrispAI as the go-to AI infrastructure layer for a specific niche — likely Robotics, Construction, or Permit/Regulatory Processing — and execute a strategic acquisition in the $5M–$10M range. The playbook: build core infrastructure, prove it with 5–10 paying POC customers, and exit to a larger AI or enterprise tech company.

## Target Verticals

| Vertical | Why |
|----------|-----|
| Robotics | Deep technical expertise; FIRST Robotics background; growing AI+robotics market |
| Construction | Underserved by AI; high permit/process complexity; on-prem data requirements |
| Permit Processing | Government/regulated workflows; strong on-prem AI fit; repeatable automation |
| Medical AI | CT/MRI imaging expertise; strict data privacy requirements fit on-prem model |

## Optimizing For

- Speed to POC customers over feature completeness
- On-prem deployment capability as core differentiator
- Niche depth over horizontal breadth
- Acquisition readiness over long-term independence

## Explicitly Not Focused On

- Large-scale VC fundraising rounds
- Broad horizontal AI platform competing with hyperscalers
- Consumer-facing products
